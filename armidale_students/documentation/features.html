Features

This is section is not complete - the following are just notes

Applications instead of web pages

armidale supports the development and deployment of applications with sophisticated graphical user interfaces. These applications are started from the armidale application launcher in much the same way that web pages are opened using web browsers.

armidale applications can be as simple as a static display of images and text (much like a basic web page) or as sophisticated as any traditional desktop application.

Universal Application Launcher

To run armidale applications, a single Java jar file needs to downloaded and run (installed). Once installed, the end user can start browsing armidale applications running around the internet or their local intranet.

The armidale launcher will run on any JDK 1.4 platform (eg. Linux, Solaris, Windows)

Radical Simplicity

no large servers and browsers, simple browser, simple conventional OO programming model, no need at all to worry about network issues.

Dynamic management of available applications

applications can be installed and removed without restarting the server

Low bandwidth open communications protocol

Communications based on simple open binary message protocol. low bandwidth, simple API to messaging, faster that XML and just as usable and open.

Platform independance

runs on any JDK 1.4 platform - linux, solaris, windows (Mac soon - when 1.4 released)

Extensible widget set

developer can add new widgets, specified in XML file. armidale will generate all of the code required to distribute new widgets. The developer only needs to implement methods used to display the widgets (using swing for example)

Applications can check for the availability of third party widgets on the client before starting.

Efficient handling of massive data sets

huge data sets can be viewed over network without sending entire data set.

Simple model for new Client Implementations

new client implementation technologies can be added (currently swing is implemented). eg. mobile devices.
